# A股核心资产基本面画像——数据分析作品集

> 基于400家A股上市公司5年财务数据，完成数据清洗、交互式仪表板搭建、AB测试统计检验和专业分析报告的完整数据分析项目。

---

## 项目背景与目标

本项目作为**数据分析岗位求职作品集**，围绕一个核心业务问题展开分析：

> **高研发投入的企业，其盈利能力（ROE）是否显著高于低研发投入的企业？**

为回答这一问题，项目构建了从**数据获取 -> 清洗 -> 指标计算 -> 可视化 -> 统计检验 -> 报告撰写**的完整分析流水线，全面展示数据分析师的四项核心能力：

| 能力维度 | 对应模块 | 展示内容 |
|---------|---------|---------|
| 数据清洗工程 | 模块一 | Pandas多表合并、缺失值填补、缩尾处理、数据质量报告 |
| 可视化与BI | 模块二 | Plotly Dash交互式仪表板（5个图表组件） |
| 统计推断 | 模块三 | PSM倾向得分匹配、Welch's T检验、Cohen's d效应量 |
| 业务洞察 | 模块四 | 3000字面向买方研究员的专业分析报告 |

---

## 数据集与来源

本项目使用**模拟生成数据**，基于真实A股上市公司财务特征设计，确保可复现性：

| 指标 | 详情 |
|------|------|
| 样本量 | **400家公司**（5个行业 x 80家） |
| 时间跨度 | **2019-2023年**（5个会计年度） |
| 行业覆盖 | 制造业、信息技术、金融业、消费品、能源 |
| 财务报表 | 利润表、资产负债表、现金流量表（共36个字段） |
| 随机种子 | `seed=2024`（NumPy `default_rng`，确保100%可复现） |

> **为什么用模拟数据？** 实际项目中可通过CSMAR/Wind/同花顺iFind获取真实数据。模拟数据的优势在于：无API依赖、可离线运行、便于面试官验证代码逻辑。

## 分析流程与方法

### 模块一：数据清洗工程

```
原始数据（含质量问题）-> 去重 -> 行业标签标准化 -> 类型转换
    -> 缺失值填补（4层策略）-> 缩尾处理 -> 清洗后宽表
```

- **缺失值填补策略**：金融业研发费填0（行业惯例）-> 时间序列前向填充 -> 行业年度中位数 -> 删除高缺失行
- **缩尾处理**：对营收、利润、资产等关键变量进行1%/99%分位截断

### 模块二：交互式仪表板

- 使用 **Plotly Dash** 构建5个交互式图表组件
- 回调机制：下拉菜单选择年份/指标/公司，图表实时更新
- 启动命令：`python 02_指标计算与可视化代码.py` -> 访问 `http://localhost:8050`

### 模块三：AB测试统计检验

```
研发强度分组 -> 平衡性检验 -> PSM倾向得分匹配（1:1）
    -> Welch's T检验 -> Cohen's d效应量 -> 稳健性检验
```

- **PSM方法**：Logistic回归估计倾向得分 + NearestNeighbors近邻匹配（卡尺=0.05）
- **稳健性检验**：替换分组阈值（前30% vs 后30%）、替换被解释变量（ROA）

### 模块四：综合分析报告

- 面向买方研究员/投资总监的3000字专业报告
- 涵盖：全市场财务画像、研发投入检验、投资启示与风险提示

## 核心结论与洞察

### 1. 全市场ROE中枢与行业分化

| 行业 | 平均ROE | 平均资产负债率 | 盈利驱动逻辑 |
|------|---------|---------------|-------------|
| 能源 | ~18% | 61% | 高波动，受大宗商品价格影响 |
| 消费品 | ~17.8% | 44% | 高周转率（0.9倍），品牌溢价 |
| 制造业 | ~12.7% | 53% | 资本密集，周期性强 |
| 金融业 | ~7.8% | 90% | 高杠杆驱动（风险敞口大） |
| 信息技术 | ~-1.9% | 33% | 高研发侵蚀利润，高毛利率（18%） |

> **关键洞察**：金融业的高ROE是通过极高杠杆（90%）实现的，风险调整后收益一般。信息技术行业因研发投入高，当期ROE被压低，但资产回报率（ROA）与低研发组无显著差异。

### 2. 研发投入的"生产力"检验

**统计检验结论**：

| 检验项目 | T统计量 | P值 | Cohen's d | 结论 |
|---------|---------|-----|-----------|------|
| **主检验（ROE）** | **-2.74** | **0.006** | **-0.14** | **高研发组ROE显著更低** |
| 稳健性检验（ROA） | 0.57 | 0.57 | 0.03 | 无显著差异 |

> **核心发现**：研发投入对**ROE有显著负向影响**，但对**ROA无显著影响**。这是因为研发费用直接减少当期净利润和所有者权益，压低ROE；但ROA的分母（总资产）更大，效应被稀释。**评估研发"真实生产力"时，ROA是比ROE更可靠的指标。**

### 3. 研发投入的门槛效应

稳健性检验（前30% vs 后30%分组）显示：
- 只有研发强度超过行业**75分位**的企业，才表现出明显的ROA优势
- 研发投入存在**规模经济**：小额研发难以形成技术壁垒

## 如何运行

### 环境配置

```bash
# 1. 克隆项目
git clone https://github.com/mangowillknow-cyber/china-a-share-fundamental-analysis.git
cd china-a-share-fundamental-analysis

# 2. 创建虚拟环境（推荐）
python -m venv venv
venv\Scripts\activate  # Windows
# source venv/bin/activate  # Mac/Linux

# 3. 安装依赖
pip install -r requirements.txt
```

### 运行各个模块

```bash
# 模块一：数据清洗（输出清洗后CSV和质量报告）
python 01_数据清洗代码.py

# 模块二：交互式仪表板（浏览器自动打开 localhost:8050）
python 02_指标计算与可视化代码.py

# 模块三：统计检验（输出 统计检验结果表.csv）
python 03_统计检验代码.py

# 数据质量报告（Jupyter Notebook）
jupyter notebook 数据质量报告.ipynb
```

### 仪表板截图预览

> 运行模块二后，在浏览器中截图保存到项目目录，并更新下方链接。

**仪表板包含5个交互组件**：

1. **KPI卡片**：全市场平均ROE/ROA/资产负债率/总资产周转率
2. **行业散点图**：X=资产负债率，Y=ROE，点大小=总资产，颜色=行业
3. **时序折线图**：各行业ROE中位数近5年走势
4. **公司雷达图**：选择任意公司，展示8维度财务画像
5. **行业热力图**：行业x年份的指标矩阵对比
